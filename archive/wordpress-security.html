I've been thinking a lot about Wordpress security recently, as I've read a number of stories about Wordpress-driven sites getting hacked. The key to not getting hacked is to be as unpredictable as possible. There are a number of patterns that occur in most Wordpress installs that can make a hackers job that much easier. Here are some key things that anyone builidng with Wordpress should be thinking about:

  • Using an administrator to post content - Use your administrator account for backend work only, especially if you are displaying your username on your site.
  • Don't create an account with the username "admin" - This is the default username, only use it if you want to be hacked.
  • Change the default wp_ table prefix - if you don't do this, hackers already know the table names in your database. Change the prefix in the wp-config.php file during the install process.
  • Define unique authentication keys and salts for your site - Generate these at the Wordpress.org secret-key service, and edit the keys in the wp-config.php file at any time.
  • Keep Wordpress up-to-date - Wordpress core developers are quick to fix bugs and vulnerabilities with regular updates, so if you're version is behind, you are leaving yourself vulnerable.
  • Create difficult passwords - This one is a no-brainer.
  • Do due diligence on any plugins that you choose to use - Many Wordpress plugins are buggy, outdated and house huge security black holes. Choose plugins wisely; make sure that they are recently updated, highly rated and well reviewed. If you are a developer, give the code a scan to make sure that you don't see any issues. Better yet, don't use a plugin when you don't really need it.
  • And remember to keep your plugins updated as well!
