Bower copies packages from Git endpoints into your project, along with their dependencies. That is all.

Burke Holland, Kendo UI

I recently started experimenting with the package manager Bower and have found it to be a useful tool in my development workflow. It is a simple tool that performs its function very well. Per the above quote, Bower simply enables you to install packages (javascript libraries, css bootstraps, etc) very quickly via the command line into your project directory. This post is a guide to getting started with Bower.

Installing Bower

You can either install Bower on it's own using npm:

$ npm install -g bower

Or it also comes pre-packaged with Yeoman, which you also install using npm:

$ npm install -g yo

The -g ensures that Bower is installed globally.

Installing Packages

Installing packages is very simple with Bower (that's the whole point, I guess). First, navigate to your project directory. Now tell Bower to install any packages you require for your project. For this example, we're going to install the packages required to build an ember application. Let's start by installing Ember itself and it's dependencies:

$ bower install ember

That's it! Have a look in your project directory, and you'll see a folder called bower_components. Inside here you'll see ember and it's two key dependencies, jquery and handlebars.

Now let's download a few other packages that we know we'll need:

$ bower install bootstrap
$ bower install underscore
$ bower install requirejs

One thing to notice is that Bower installs the latest versions of these libraries by default. But let's say we need to support IE8 with our project. jQuery dropped support for IE8 with v2.0, so we will need version 1.9 for this project. With Bower, simply install the version that you need and it will overwrite the current version installed:

$ bower install jquery#1.9.1

To see a list of the packages you have installed, in a handy dependency tree view, use the command bower ls:

$ bower ls

bower check-new     Checking for new versions of the project dependencies..
├─┬ bootstrap#3.0.0 extraneous
│ └── jquery#1.9.1 (2.0.3 available)
├─┬ ember#1.0.0 extraneous
│ ├── handlebars#1.0.0
│ └── jquery#1.9.1 (2.0.3 available)
├── jquery#1.9.1 extraneous (latest is 2.0.3)
├── requirejs#2.1.8 extraneous
└── underscore#1.5.2 extraneous

Finding Packages

If you don't know the exact name of the package you are trying to download, or simply want to search for which packages are available, Bower has a package registry that you can search from the command line.

$ bower search [package name]

Create a Bower json file

Once you have all your packages installed, it's time to create a bower.json file for your project. You do this by simply running:

$ bower init

Unless you want to add additional information, hit return several times to use the default options for the json file. Once the initialization is complete, you'll see a bower.json file in your project's root directory. The important thing to note in this file is the list of all the dependencies for your project. If you wanted to create a similar project with the same dependencies, you would simply copy this json file into a new project directory, and run $ bower install. Bower would sutomatically download all the dependencies for your new project.

Changing the bower_components directory

If you are like me, you won't like the fact that Bower saves the installed packages into a directory called bower_components. I personally like these packages to be included in a js/libs directory. This is simple to achieve.

First, create a file named .bowerrc in your project's root directory (note, the . prefix means this is a hidden file).

Add the following code to your .bowerrc file:

{
    "directory" : "js/libs"
}

Now run the following command and your directory structure will be updated. Note that the bower_components folder will remain in your project directory, but it is now safe to delete it.

$ bower install

That's all folks

As you can see, Bower is a powerful little tool that can help streamline your workflow for setting up new projects. There is plenty of information not covered here such as registering your own packages, but this should be enough info to get you started.
